BMF-CSS 開発者ガイド

BMF-CSSドキュメントの開発・メンテナンス方針

ドキュメント作成の基本方針

BMF-CSSのみ使用

  • BMF-CSSで用意されているCSSのみを使用
  • インラインスタイルの追加禁止
  • ドキュメント専用のclass・id定義の追加禁止
  • 追加スタイルが必要な場合は事前に相談

ドキュメント構造

  • 複数ページ構成：各セクション・コンポーネントごとに別ページ
  • ナビゲーション：サイドバー方式（BMF-CSSの機能を使用）
  • 言語：英語のみ

ドキュメントアウトライン

1. 概要（Overview）

  • フレームワークの紹介
  • 特徴と利点
  • 設計思想（セマンティック + ユーティリティのハイブリッド設計）

2. インストールと使用方法

  • NPM/Yarnでのインストール方法
  • CDNからの利用方法
  • プロジェクトへの組み込み方法

3. 実装アプローチ

  • CSS-Based実装（JavaScriptなし）の概要と利点
  • Hybrid実装（CSS + JavaScript）の概要と利点
  • Attribute-Based実装（データ属性による宣言的実装）の概要と利点
  • 実装パターンの選択ガイド

4. Base

  • リセットとノーマライズ
  • ルート要素（html, body）
  • タイポグラフィ基本設定
  • リンクスタイル
  • フォームリセット
  • コード例

5. Tokens

  • カラーシステム
  • タイポグラフィ変数
  • スペーシング
  • シャドウとエレベーション
  • ブレークポイント
  • CSSカスタム変数一覧
  • コード例

6. Components

各コンポーネントについて、3つの実装パターンのコード例と説明：

  • ボタン
  • フォーム要素
  • カード
  • ナビゲーション
  • アラート/通知
  • モーダル
  • ツールチップ
  • タブ
  • テーブル
  • ページネーション
  • その他のUIコンポーネント

7. Utilities

  • スペーシング（マージン、パディング）
  • テキストスタイル
  • 表示プロパティ
  • 色と背景
  • ボーダーとシャドウ
  • フレックスとグリッド
  • ポジショニング
  • コード例と使用パターン

8. Themes

  • デフォルトテーマ
  • ダークモード（3つの実装パターン）
  • テーマのカスタマイズ方法

9. デモ

実用的なレベルのデモ（各実装パターン別）：

表示方法の方針

コード例

  • BMF-CSSが提供するコードブロック表示機能を使用
  • 実装と表示を上下に配置
  • 実装パターンはBMF-CSSのタブUIで切り替え（CSS-Based → Hybrid → Attribute-Basedの順）

レスポンシブ対応

  • BMF-CSSが提供するレスポンシブ機能のみを使用
  • モバイル向けナビゲーションはBMF-CSSの既存機能で対応

ページ共通要素

  • ヘッダー/フッター（BMF-CSSの既存コンポーネントを使用）
  • サイドバーナビゲーション（BMF-CSSのナビゲーションコンポーネントを使用）

コンポーネント表示

  • BMF-CSSが提供するレイアウトコンポーネントのみを使用して配置

重要：どうしても追加スタイルが必要な場合（例：コードハイライト）は、事前に相談の上で対応を決定します。
