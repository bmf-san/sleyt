BMF-CSSドキュメントの開発・メンテナンス方針

ドキュメント作成の基本方針

BMF-CSSのみ使用

  • BMF-CSSで用意されているCSSのみを使用
  • インラインスタイルの追加禁止
  • ドキュメント専用のclass・id定義の追加禁止

ドキュメント構造

  • 複数ページ構成：各セクション・コンポーネントごとに別ページ
  • ナビゲーション：サイドバー方式（BMF-CSSの機能を使用）
  • 言語：英語のみ

表示方法の方針

コード例

  • BMF-CSSが提供するコードブロック表示機能を使用
  • 実装と表示を上下に配置
  • 実装パターンはBMF-CSSのタブUIで切り替え（CSS-Based → Hybrid → Attribute-Basedの順）

レスポンシブ対応

  • BMF-CSSが提供するレスポンシブ機能のみを使用
  • モバイル向けナビゲーションはBMF-CSSの既存機能で対応

ページ共通要素

  • ヘッダー/フッター（BMF-CSSの既存コンポーネントを使用）
  • サイドバーナビゲーション（BMF-CSSのナビゲーションコンポーネントを使用）

コンポーネント表示

  • BMF-CSSが提供するレイアウトコンポーネントのみを使用して配置

重要：どうしても追加スタイルが必要な場合（例：コードハイライト）は、事前に相談の上で対応を決定します。
