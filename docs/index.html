Overview

BMF-CSS Framework

BMF-CSS is a lightweight, flexible CSS framework designed for modern web applications. It provides a set of carefully crafted components and utilities that help you build consistent and responsive interfaces with minimal effort.

Features and Benefits

  • Semantic component classes with utility classes for fine-tuning
  • Consistent design system with color scales and typography rules
  • Responsive utilities with simple prefixes
  • Lightweight and performant with minimal CSS
  • Easily customizable with CSS custom properties

Design Philosophy

BMF-CSS follows a hybrid approach combining semantic components and utility classes. This provides a balance between structure and flexibility, allowing you to create complex layouts without sacrificing maintainability.

Installation & Usage

Installing BMF-CSS

NPM Installation

npm install bmf-css

Yarn Installation

yarn add bmf-css

CDN Usage

<link rel="stylesheet" href="https://cdn.example.com/bmf-css/1.0.0/bmf-css.min.css">

Integrating with Your Project

Import in JavaScript

import 'bmf-css/dist/bmf-css.min.css';

Import in CSS/SCSS

@import 'bmf-css/dist/bmf-css.min.css';

Implementation Approach

BMF-CSS uses a CSS-Based implementation approach, providing a simple and intuitive way to style your application.

CSS-Based Implementation

The CSS-based approach is the most straightforward way to use BMF-CSS. Simply apply CSS classes to your HTML elements.

Benefits:

  • No JavaScript dependencies
  • Simple and intuitive
  • Works in all environments
  • Easy to maintain
<button class="btn-primary">Click Me</button>

Implementation Best Practices

  • Use semantic HTML elements with appropriate classes
  • Combine component classes with utility classes for fine-tuning
  • Keep your HTML clean and maintainable
  • Use the responsive prefix modifiers for different screen sizes

Usage Guidelines:

  • 50-100: Backgrounds, separator elements
  • 200-300: Borders, dividers, inactive elements
  • 400-500: Placeholders, secondary text
  • 600-700: Body text, control elements
  • 800-950: Headings, emphasized text, dark mode backgrounds

Accessibility Note:

Color should never be used as the only means of conveying information. Combine with text or shapes for users with visual impairments.

Text-background contrast ratio requirements:

  • WCAG AA level: 4.5:1+ for normal text, 3:1+ for large text

Dark Mode

BMF-CSS provides a simple CSS-Based approach to implementing dark mode using a class on the body element. 

/* Define dark theme colors */
.dark-theme {
  --background: 222.2 84% 4.9%;
  --foreground: 210 40% 98%;

  --card: 222.2 84% 4.9%;
  --card-foreground: 210 40% 98%;

  --primary: 210 40% 98%;
  --primary-foreground: 222.2 47.4% 11.2%;

  --secondary: 217.2 32.6% 17.5%;
  --secondary-foreground: 210 40% 98%;

  --accent: 217.2 32.6% 17.5%;
  --accent-foreground: 210 40% 98%;

  --muted: 217.2 32.6% 17.5%;
  --muted-foreground: 215 20.2% 65.1%;

  --destructive: 0 62.8% 30.6%;
  --destructive-foreground: 210 40% 98%;

  --border: 217.2 32.6% 17.5%;
}

To implement the dark mode, simply add the dark-theme class to the body element: 

<body class="dark-theme">
  <!-- Your content will use dark theme styles -->
</body>

<style>

  }

Customizing Themes

You can customize BMF-CSS themes by overriding CSS variables in your own stylesheet. 

/* Create a custom theme */
:root {
  /* Override primary color */
  --primary: 262.1 83.3% 57.8%; /* Purple */
  --primary-foreground: 210 40% 98%;

  /* Override border radius */
  --radius: 0.25rem;
}

/* Create a custom dark theme */
.dark-theme {
  /* Override dark theme background */
  --background: 220 20% 10%;
  --foreground: 210 40% 98%;
}
