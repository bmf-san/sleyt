Overview

BMF-CSS Framework

BMF-CSS is a lightweight, flexible CSS framework designed for modern web applications. It provides a set of carefully crafted components and utilities that help you build consistent and responsive interfaces with minimal effort.

Features and Benefits

  • Semantic component classes with utility classes for fine-tuning
  • Consistent design system with color scales and typography rules
  • Responsive utilities with simple prefixes
  • Lightweight and performant with minimal CSS
  • Easily customizable with CSS custom properties

Design Philosophy

BMF-CSS follows a hybrid approach combining semantic components and utility classes. This provides a balance between structure and flexibility, allowing you to create complex layouts without sacrificing maintainability.

Installation & Usage

Installing BMF-CSS

NPM Installation

npm install bmf-css

Yarn Installation

yarn add bmf-css

CDN Usage

<link rel="stylesheet" href="https://cdn.example.com/bmf-css/1.0.0/bmf-css.min.css">

Integrating with Your Project

Import in JavaScript

import 'bmf-css/dist/bmf-css.min.css';

Import in CSS/SCSS

@import 'bmf-css/dist/bmf-css.min.css';

Basic HTML Template

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My BMF-CSS Project</title>
  <link rel="stylesheet" href="path/to/bmf-css.min.css">
</head>
<body>
  <div class="container">
    <h1 class="text-2xl mt-4">Hello, BMF-CSS!</h1>
    <button class="btn-primary">Click Me</button>
  </div>
</body>
</html>

Implementation Approaches

BMF-CSS offers three implementation approaches, giving you flexibility to choose the best method for your project's needs.

CSS-Based Implementation

The CSS-based approach is the most straightforward way to use BMF-CSS. Simply apply CSS classes to your HTML elements.

Benefits:

  • No JavaScript dependencies
  • Simple and intuitive
  • Works in all environments
  • Easy to maintain
<button class="btn-primary">Click Me</button>

Hybrid Implementation (CSS + JS)

The hybrid approach combines CSS classes with JavaScript enhancements for more interactive components.

Benefits:

  • Enhanced interactivity
  • Dynamic state management
  • Progressive enhancement
  • JavaScript APIs for complex interactions
<button class="btn-primary" data-toggle="modal" data-target="#my-modal">
  Open Modal
</button>

<script>
  // JavaScript enhancement
  document.querySelector('[data-toggle="modal"]').addEventListener('click', function() {
    const targetId = this.getAttribute('data-target');
    const modal = document.querySelector(targetId);
    modal.classList.add('active');
  });
</script>

Attribute-Based Implementation

The attribute-based approach uses HTML data attributes instead of classes for styling.

Benefits:

  • Cleaner HTML with separation of styling and structure
  • More semantic markup
  • Easier to manage in component-based systems
  • Better for complex components with multiple states
<button data-bmf-button="primary">Click Me</button>

Choosing the Right Approach

Factor CSS-Based Hybrid Attribute-Based
Simplicity High Medium Medium
Interactivity Low High Medium
Maintainability Medium Medium High
Best For Static sites Web applications Component systems

Components

BMF-CSS provides a comprehensive set of UI components that can be used to build modern web interfaces. Each component is available in three implementation approaches: CSS-Based, Hybrid, and Attribute-Based.

Buttons

Buttons are used to trigger actions or events, such as submitting a form, opening a dialog, canceling an action, or performing a delete operation. 

<button class="btn-primary">Primary Button</button>
<button class="btn-secondary">Secondary Button</button>
<button class="btn-outline">Outline Button</button>
<button class="btn-ghost">Ghost Button</button>
<button class="btn-destructive">Destructive Button</button>
<button class="btn-primary" data-loading="false" id="loading-button">
  Click to Load
</button>

<script>
  document.getElementById('loading-button').addEventListener('click', function() {
    this.setAttribute('data-loading', 'true');
    this.classList.add('btn-loading');

    // Simulate loading
    setTimeout(() => {
      this.setAttribute('data-loading', 'false');
      this.classList.remove('btn-loading');
    }, 2000);
  });
</script>

<div class="card" data-collapsible>
  <div class="card-header" data-toggle="collapse">
    <h3 class="card-title">Collapsible Card</h3>
    <button class="btn-icon">
      <svg width="16" height="16" viewBox="0 0 24 24">
        <path d="M7 10l5 5 5-5H7z"></path>
      </svg>
    </button>
  </div>
  <div class="card-body collapse">
    <p>This content can be collapsed and expanded.</p>
  </div>
</div>

<script>
  document.querySelectorAll('[data-toggle="collapse"]').forEach(toggle => {
    toggle.addEventListener('click', function() {
      const target = this.nextElementSibling;
      target.classList.toggle('show');
    });
  });
</script>
