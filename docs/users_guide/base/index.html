Base

Foundation styles that form the basis of BMF-CSS

Overview

Base styles in BMF-CSS provide a solid foundation for your project by normalizing browser inconsistencies and establishing sensible defaults for HTML elements. These styles ensure a consistent starting point across different browsers and devices.

Base Style Categories

Reset & Normalize

Standardizes default styling across browsers by removing inconsistencies and providing a clean foundation.

View Reset

Typography

Establishes base styles for text elements including headings, paragraphs, lists, and more.

View Typography

Links

Defines styles for hyperlinks, including default, hover, focus, and active states.

View Links

Forms

Sets default styles for form elements such as inputs, selects, textareas, and buttons.

View Forms

Key Features

Lightweight Reset

BMF-CSS uses a lightweight reset approach that normalizes styles without completely stripping them. This provides a clean foundation while maintaining useful browser defaults.

Box Sizing

All elements use box-sizing: border-box by default, making layout calculations more intuitive since padding and borders are included in the element's width and height.

Responsive Typography

Base typography uses relative units (rem/em) rather than pixels, ensuring text scales appropriately with user preferences and device settings.

Focus States

Accessible focus states are defined for interactive elements, ensuring keyboard navigation is properly supported.

Form Element Consistency

Form elements are normalized across browsers while maintaining their native functionality and accessibility features.

Base CSS Structure

Base styles in BMF-CSS are organized into the following files:

src/
└── base/
    ├── _reset.css        # Browser normalization and resets
    ├── _typography.css   # Text styling defaults
    ├── _links.css        # Link styling
    ├── _forms.css        # Form element styling
    └── index.css         # Main import file

How Base Styles Work

Base styles are automatically included when you import BMF-CSS. They apply globally to HTML elements without the need for additional classes. For example:

HTML Example

<h1>Page Title</h1>
<p>This is a paragraph with a <a href="#">link</a>.</p>
<form>
  <label for="name">Name:</label>
  <input type="text" id="name">
  <button type="submit">Submit</button>
</form>

These elements will be styled with BMF-CSS base styles automatically, providing a clean, consistent appearance without requiring additional classes.

Customizing Base Styles

You can customize base styles by overriding CSS variables or directly targeting elements in your own stylesheets:

Using CSS Variables

:root {
  --body-font-family: 'Your Font', sans-serif;
  --body-line-height: 1.6;
  --heading-font-family: 'Your Heading Font', serif;
  --link-color: #0066cc;
  --link-hover-color: #004499;
}

Direct Override

/* Your custom.css, loaded after BMF-CSS */
body {
  font-family: 'Your Font', sans-serif;
  line-height: 1.6;
}

a {
  color: #0066cc;
  text-decoration: none;
}

a:hover {
  color: #004499;
  text-decoration: underline;
}
