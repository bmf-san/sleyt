Implementation Approaches

BMF-CSS provides three ways to implement buttons:

CSS-Based Implementation

The CSS-Based implementation uses only CSS classes to style and define button behavior:

Example

Code

<button class="btn btn-primary">Click Me</button>

/* HTML anchor as button */
<a href="#" class="btn btn-primary">Link Button</a>

/* Input as button */
<input type="submit" class="btn btn-primary" value="Submit">

Advantages

  • No JavaScript required
  • Simple implementation
  • Works with any element that can be styled as a button

Hybrid Implementation

The Hybrid implementation combines CSS styling with JavaScript functionality for more interactive buttons:

Example

HTML

<button class="btn btn-primary" id="js-animated-button">Click Me</button>

JavaScript

document.addEventListener('DOMContentLoaded', function() {
  const button = document.getElementById('js-animated-button');

  if (button) {
    button.addEventListener('click', function() {
      // Add animation class
      this.classList.add('btn-clicked');

      // Show the ripple effect
      const ripple = document.createElement('span');
      ripple.classList.add('btn-ripple');
      this.appendChild(ripple);

      // Remove animation after completion
      setTimeout(() => {
        this.classList.remove('btn-clicked');
        ripple.remove();
      }, 600);
    });
  }
});

Advantages

  • Enhanced animations and effects
  • Better user feedback
  • Custom behaviors on various events

Attribute-Based Implementation

The Attribute-Based implementation uses HTML data attributes to define button properties and behavior:

Example

HTML

<button data-btn="primary">Click Me</button>

<button data-btn="outline" data-btn-size="lg" data-btn-state="loading">Loading...</button>

<button data-btn="danger" data-btn-block>Delete Account</button>

JavaScript

document.addEventListener('DOMContentLoaded', function() {
  // Initialize all attribute-based buttons
  initButtons();

  function initButtons() {
    const buttons = document.querySelectorAll('[data-btn]');

    buttons.forEach(button => {
      // Add base class
      button.classList.add('btn');

      // Add variant class
      const variant = button.getAttribute('data-btn');
      if (variant) {
        button.classList.add(`btn-${variant}`);
      }

      // Add size class if specified
      const size = button.getAttribute('data-btn-size');
      if (size) {
        button.classList.add(`btn-${size}`);
      }

      // Add state class if specified
      const state = button.getAttribute('data-btn-state');
      if (state) {
        button.classList.add(state);
      }

      // Add block class if specified
      if (button.hasAttribute('data-btn-block')) {
        button.classList.add('btn-block');
      }
    });
  }
});

Advantages

  • Declarative approach
  • Cleaner HTML markup
  • Easier to maintain and update
  • More intuitive configuration

Accessibility Considerations

When implementing buttons, consider these accessibility best practices:

  • Use the <button> element for actions that don't navigate to a new page
  • Use <a> elements for links that navigate to a new page or anchor
  • Provide descriptive text that explains the button's purpose
  • Ensure buttons have sufficient color contrast
  • Add aria-label for icon-only buttons
  • Use aria-disabled="true" instead of the disabled attribute when you want the button to remain focusable

Example with Accessibility Features

<!-- Icon-only button with accessible label -->
<button class="btn btn-icon btn-primary" aria-label="Add new item">
  <span class="icon-add"></span>
</button>

<!-- Disabled but focusable button -->
<button class="btn btn-primary disabled" aria-disabled="true">
  Submit
</button>

  --btn-primary-text: #ffffff;
  --btn-primary-border: #2b6cb0;
  --btn-border-radius: 0.25rem;
  --btn-font-weight: 600;
  --btn-padding-x: 1rem;
  --btn-padding-y: 0.5rem;
}
