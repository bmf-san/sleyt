Implementation Approaches

Each component in BMF-CSS can be implemented using one of three approaches:

CSS-Based Implementation

Uses only CSS classes to style and define component behavior. No JavaScript required.

Learn More

Hybrid Implementation

Combines CSS styling with JavaScript functionality for more interactive components.

Learn More

Attribute-Based Implementation

Uses HTML data attributes to define component properties and behavior.

Learn More

Component Structure

Most BMF-CSS components follow a consistent naming pattern:

Base Class

The main class that identifies the component type (e.g., btn, card, modal).

Variation Modifiers

Classes that modify the appearance or behavior of the component (e.g., btn-primary, card-bordered).

State Classes

Classes that represent different states of the component (e.g., active, disabled, loading).

Size Modifiers

Classes that control the size of the component (e.g., btn-sm, btn-lg).
