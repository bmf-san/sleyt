Design Tokens

基本的なデザイン単位と変数のシステム

概要

デザイントークンはBMF-CSSのデザインシステムの基礎を形成する変数です。色、タイポグラフィ、間隔、影などの基本的なデザイン要素を定義し、一貫したユーザーインターフェイスを作成するために使用されます。

すべてのトークンはCSS変数（カスタムプロパティ）として実装されており、アプリケーション全体での簡単な再利用とカスタマイズが可能です。

トークンカテゴリー

カラー

プライマリ、セカンダリ、アクセント色、さらにエラー、警告、成功状態用の色を含む基本カラーパレット。

カラーを見る

タイポグラフィ

フォントファミリー、フォントサイズ、行の高さ、フォントウェイトなどのテキスト関連の変数。

タイポグラフィを見る

間隔

マージン、パディング、ギャップなどのレイアウト間隔に使用される一貫した間隔の値。

間隔を見る

影と高さ

様々なレベルのエレベーションや深度を表現するボックスシャドウの値。

影を見る

ブレークポイント

レスポンシブデザインをサポートするためのメディアクエリブレークポイント。

ブレークポイントを見る

トークンの使用方法

BMF-CSSのデザイントークンはCSS変数として実装されており、次のように簡単に使用できます：

基本的な使用例

.my-element {
  /* カラートークンの使用 */
  color: var(--color-primary);
  background-color: var(--color-background);

  /* 間隔トークンの使用 */
  margin: var(--spacing-md);
  padding: var(--spacing-sm) var(--spacing-md);

  /* タイポグラフィトークンの使用 */
  font-family: var(--font-family-base);
  font-size: var(--font-size-md);
  line-height: var(--line-height-normal);

  /* 影トークンの使用 */
  box-shadow: var(--shadow-md);
}

トークンファイル構造

BMF-CSSのトークンは以下のように構造化されています：

src/
└── tokens/
    ├── _colors.css       # カラートークン
    ├── _typography.css   # タイポグラフィトークン
    ├── _spacing.css      # 間隔トークン
    ├── _shadows.css      # 影と高さトークン
    ├── _breakpoints.css  # ブレークポイントトークン
    └── index.css         # メインインポートファイル

トークンのカスタマイズ

プロジェクトでBMF-CSSデザイントークンをカスタマイズするには、以下の方法があります：

1. CSSファイルでの上書き

独自のCSSファイルでトークン値を上書きします（BMF-CSSをロードした後に読み込む）：

/* custom-tokens.css */
:root {
  --color-primary: #1e88e5;
  --color-secondary: #5e35b1;
  --font-family-base: 'Roboto', sans-serif;
  --spacing-base: 0.5rem;
}

2. インラインでの上書き

特定の要素またはコンポーネントスコープでトークンを上書きします：

.custom-component {
  --color-primary: #ff5722;
  --spacing-md: 1.25rem;
}

3. テーマの使用

BMF-CSSのテーマシステムを使用して複数のトークン値セットを切り替えます：

/* テーマを適用するには、ルート要素にテーマクラスを追加します */
<html class="theme-dark">
  ...
</html>

詳細なテーマ設定については、テーマカスタマイズセクションを参照してください。

トークンとコンポーネントの関係

BMF-CSSのコンポーネントはすべて基盤となるデザイントークンを使用して構築されています。これにより、以下の利点があります：

  • 一貫性： すべてのコンポーネントが同じ基本的なデザイン言語を共有
  • 保守性： 基盤となるトークンを更新することで、すべてのコンポーネントが自動的に更新される
  • テーマ対応： トークンを変更するだけで、アプリケーション全体のテーマを簡単に切り替え可能
  • カスケード活用： 特定のコンポーネントやコンテキストでトークンを選択的に上書き可能

例えば、ボタンコンポーネントは次のようにトークンを使用しています：

.btn {
  /* トークンベースのスタイル */
  font-family: var(--font-family-base);
  font-size: var(--font-size-md);
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--border-radius-md);
}

.btn-primary {
  background-color: var(--color-primary);
  color: var(--color-white);
}

.btn-secondary {
  background-color: var(--color-secondary);
  color: var(--color-white);
}
