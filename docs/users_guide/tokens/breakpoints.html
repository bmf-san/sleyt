ブレークポイント

レスポンシブデザインのためのブレークポイント定義とその活用方法

概要

BMF-CSSは、さまざまな画面サイズやデバイスに対応したレスポンシブデザインを実現するために、一連の標準ブレークポイントを提供しています。これらのブレークポイントを活用することで、画面サイズに応じてレイアウトやコンポーネントの表示を調整できます。

標準ブレークポイント

BMF-CSSでは、以下の標準ブレークポイントを使用しています：

ブレークポイント名 最小幅 説明
sm 640px 小型デバイス（タブレット、大型モバイル）
md 768px 中型デバイス（タブレット）
lg 1024px 大型デバイス（デスクトップ）
xl 1280px 特大デバイス（ワイドスクリーン）
2xl 1536px 超大型デバイス（大型ディスプレイ）

注： BMF-CSSはモバイルファーストのアプローチを採用しています。つまり、スタイルはまず小さな画面サイズ向けに定義され、その後大きな画面サイズに合わせて調整されます。

コンテナのブレークポイント

BMF-CSSの.containerクラスは、以下のブレークポイントに基づいて自動的に幅を調整します：

ブレークポイント コンテナの最大幅
< 576px 100%
≥ 576px 540px
≥ 768px 720px
≥ 992px 960px
≥ 1200px 1140px

コンテナバリエーション

  • .container - 標準コンテナ（すべてのブレークポイントで応答）
  • .container-fluid - 常に100%幅を維持するコンテナ
  • .container-sm - 576px以上で最大幅が固定されるコンテナ
  • .container-md - 768px以上で最大幅が固定されるコンテナ
  • .container-lg - 992px以上で最大幅が固定されるコンテナ
  • .container-xl - 1200px以上で最大幅が固定されるコンテナ

レスポンシブユーティリティ

BMF-CSSでは、ブレークポイント接頭辞を使用して、特定の画面サイズでのみスタイルを適用できます：

レスポンシブ接頭辞の使用例

<!-- モバイルでは縦並び、768px以上では横並びになる -->
<div class="flex-col md:flex-row">
  <div class="p-4">アイテム1</div>
  <div class="p-4">アイテム2</div>
</div>

<!-- モバイルでは非表示、768px以上で表示される -->
<div class="hidden md:block">
  大きな画面でのみ表示されるコンテンツ
</div>

<!-- グリッドカラムがブレークポイントに応じて変化 -->
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3">
  <div class="p-4">アイテム1</div>
  <div class="p-4">アイテム2</div>
  <div class="p-4">アイテム3</div>
</div>

独自のメディアクエリ

BMF-CSSのブレークポイントを独自のCSSで使用する場合は、以下のメディアクエリを使用できます：

/* Small (sm) */
@media (min-width: 640px) {
  /* スタイルをここに記述 */
}

/* Medium (md) */
@media (min-width: 768px) {
  /* スタイルをここに記述 */
}

/* Large (lg) */
@media (min-width: 1024px) {
  /* スタイルをここに記述 */
}

/* Extra Large (xl) */
@media (min-width: 1280px) {
  /* スタイルをここに記述 */
}

/* 2X Large (2xl) */
@media (min-width: 1536px) {
  /* スタイルをここに記述 */
}

レスポンシブデザインのベストプラクティス

1. モバイルファーストアプローチを採用する

まずモバイル向けのスタイルを定義し、その後大きなブレークポイント向けにスタイルを拡張します。

2. コンテナを使用する

.containerクラスを使用して、コンテンツを適切な幅に制限します。

3. 複雑なレイアウトにはグリッドシステムを使用する

複数カラムのレイアウトには、レスポンシブなグリッドシステムを活用します。

4. レスポンシブユーティリティを適切に使用する

特定の画面サイズで要素を表示/非表示にするために、hidden/blockなどのユーティリティクラスを組み合わせます。

5. デバイスでテストする

実際のデバイスや、ブラウザの開発者ツールのデバイスエミュレーションモードでデザインをテストします。

レスポンシブイメージ

BMF-CSSでは、画像を自動的にレスポンシブにするスタイルが適用されています：

img {
  max-width: 100%;
  height: auto;
}

これにより、画像はコンテナの幅を超えないようになります。より高度な制御が必要な場合は、以下のユーティリティクラスを使用できます：

  • .img-fluid - レスポンシブイメージ（max-width: 100%, height: auto）
  • .img-full - 親要素いっぱいに広がるイメージ（width: 100%）
  • .img-cover - 親要素全体をカバーするイメージ（object-fit: cover）
  • .img-contain - 親要素内に収まるイメージ（object-fit: contain）
