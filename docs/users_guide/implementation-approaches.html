Implementation Approaches

Three ways to implement BMF-CSS components

Overview

BMF-CSS provides three distinct implementation patterns for its components, each with its own advantages. Choose the approach that best fits your project requirements and preferences.

CSS-Based Implementation

Description

The CSS-Based implementation uses only CSS to create components, without requiring any JavaScript. This approach relies on CSS selectors, pseudo-classes, and clever structure to create interactive elements.

Key Features:

  • No JavaScript required
  • Lightweight and fast
  • Works without any build process
  • SEO-friendly
  • Minimal dependencies

Best For:

  • Static sites
  • Projects where JavaScript should be minimized
  • Simple interactive components
  • Maximum performance optimization

Example: Dropdown Menu

Here's how to implement a dropdown menu using the CSS-Based approach:

<div class="dropdown">
  <button class="btn dropdown-toggle">Dropdown</button>
  <ul class="dropdown-menu">
    <li><a href="#" class="dropdown-item">Option 1</a></li>
    <li><a href="#" class="dropdown-item">Option 2</a></li>
    <li><a href="#" class="dropdown-item">Option 3</a></li>
  </ul>
</div>

Hybrid Implementation

Description

The Hybrid implementation combines CSS styling with JavaScript functionality. This approach leverages CSS for appearance and basic interactions, while using JavaScript for more complex behavior and state management.

Key Features:

  • Enhanced functionality through JavaScript
  • Better control over component states
  • Improved accessibility
  • More complex interactions possible
  • Programmatic control over components

Best For:

  • Complex interactive components
  • Applications requiring state management
  • Components needing dynamic content
  • Projects where accessibility is a priority

Example: Dropdown Menu

Here's how to implement a dropdown menu using the Hybrid approach:

<div class="dropdown" id="js-dropdown">
  <button class="btn dropdown-toggle" id="js-dropdown-toggle" aria-haspopup="true" aria-expanded="false">Dropdown</button>
  <ul class="dropdown-menu" id="js-dropdown-menu" aria-labelledby="js-dropdown-toggle">
    <li><a href="#" class="dropdown-item">Option 1</a></li>
    <li><a href="#" class="dropdown-item">Option 2</a></li>
    <li><a href="#" class="dropdown-item">Option 3</a></li>
  </ul>
</div>
document.addEventListener('DOMContentLoaded', function() {
  const dropdownToggle = document.getElementById('js-dropdown-toggle');
  const dropdownMenu = document.getElementById('js-dropdown-menu');

  if (dropdownToggle && dropdownMenu) {
    dropdownToggle.addEventListener('click', function() {
      const expanded = this.getAttribute('aria-expanded') === 'true';

      // Toggle dropdown visibility
      dropdownMenu.classList.toggle('show');

      // Update ARIA attributes
      this.setAttribute('aria-expanded', !expanded);
    });

    // Close dropdown when clicking outside
    document.addEventListener('click', function(event) {
      if (!dropdownToggle.contains(event.target) && !dropdownMenu.contains(event.target)) {
        dropdownMenu.classList.remove('show');
        dropdownToggle.setAttribute('aria-expanded', 'false');
      }
    });

    // Add keyboard navigation
    dropdownMenu.addEventListener('keydown', function(event) {
      if (event.key === 'Escape') {
        dropdownMenu.classList.remove('show');
        dropdownToggle.setAttribute('aria-expanded', 'false');
        dropdownToggle.focus();
      }
    });
  }
});

Attribute-Based Implementation

Description

The Attribute-Based implementation uses HTML data attributes to initialize and configure components. This declarative approach helps separate functionality from presentation and makes the code easier to understand and maintain.

Key Features:

  • Declarative implementation
  • Cleaner HTML
  • Better separation of concerns
  • Easier component customization
  • Improved maintainability

Best For:

  • Larger projects with many developers
  • Components with many configuration options
  • Projects requiring clear organization
  • When consistency is a priority

Example: Dropdown Menu

Here's how to implement a dropdown menu using the Attribute-Based approach:

<div data-dropdown>
  <button class="btn" data-dropdown-toggle>Dropdown</button>
  <ul class="dropdown-menu" data-dropdown-menu>
    <li><a href="#" class="dropdown-item">Option 1</a></li>
    <li><a href="#" class="dropdown-item">Option 2</a></li>
    <li><a href="#" class="dropdown-item">Option 3</a></li>
  </ul>
</div>
document.addEventListener('DOMContentLoaded', function() {
  // Initialize all dropdowns on the page
  initDropdowns();

  function initDropdowns() {
    const dropdowns = document.querySelectorAll('[data-dropdown]');

    dropdowns.forEach(dropdown => {
      const toggle = dropdown.querySelector('[data-dropdown-toggle]');
      const menu = dropdown.querySelector('[data-dropdown-menu]');

      if (toggle && menu) {
        // Add ARIA attributes
        toggle.setAttribute('aria-haspopup', 'true');
        toggle.setAttribute('aria-expanded', 'false');

        // Add event listeners
        toggle.addEventListener('click', function() {
          const expanded = this.getAttribute('aria-expanded') === 'true';

          // Toggle dropdown visibility
          menu.classList.toggle('show');

          // Update ARIA attributes
          this.setAttribute('aria-expanded', !expanded);
        });

        // Close dropdown when clicking outside
        document.addEventListener('click', function(event) {
          if (!toggle.contains(event.target) && !menu.contains(event.target)) {
            menu.classList.remove('show');
            toggle.setAttribute('aria-expanded', 'false');
          }
        });
      }
    });
  }
});

Choosing the Right Approach

Consider these factors when choosing which implementation pattern to use for your project:

Factor CSS-Based Hybrid Attribute-Based
Complexity Low Medium Medium-High
Performance Excellent Good Good
Accessibility Basic Excellent Excellent
Interactivity Limited Extensive Extensive
Maintainability Good Good Excellent
Learning Curve Low Medium Medium

Mixed Approach

You're not limited to using just one implementation pattern throughout your project. In fact, it's common and often beneficial to mix approaches based on the specific requirements of each component:

  • Use CSS-Based for simple UI elements like buttons, cards, and basic dropdowns
  • Use Hybrid for components requiring complex interactions like modals, tabs, and sliders
  • Use Attribute-Based for components that need to be reused extensively or where configuration options are important

This flexible approach allows you to optimize for both simplicity and functionality where appropriate.
